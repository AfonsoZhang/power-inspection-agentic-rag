# PRD v1.0 - 无人机巡检多模态 RAG 报告生成系统

> 版本: v1.0
> 撰写人: 张叔阳
> 评审日期: 2026-04-20
> 状态: 内部 PoC（Proof of Concept）

## 一、背景与机会

### 1.1 业务背景

输电线路 / 工业设备的无人机巡检过去 5 年快速普及，单次巡检产出 100-1000 张影像，但下游环节仍高度依赖人工：

- 工程师对照规程逐张判定，平均 3-5 分钟 / 张
- 历史相似案例无法复用，相同缺陷的处置经验无沉淀
- 巡检报告人工撰写，单份耗时 1-2 小时
- 跨次巡检对比缺乏自动化能力

### 1.2 机会判断

| 触发条件 | 当前是否成立 |
|---|---|
| 大模型成本下降到企业可承受 | DeepSeek-V3 输入价格已降至 1 元 / 百万 token |
| 多模态模型可识别工业场景 | VLM 多模态模型在缺陷描述类任务可用 |
| 企业有数字化基础（资产档案 / 历史报告） | 多数能源 / 制造企业已具备 |
| 业务方有降本压力 | 国网 2025 年明确"运维降本提质" |

结论：**RAG + 多模态在巡检场景具备落地条件**。

## 二、目标用户与场景

### 2.1 用户分层

| 角色 | 占比 | 核心痛点 | 期望解决 |
|---|---|---|---|
| 一线巡检员（新手） | 40% | 看不懂复杂缺陷、不熟悉规程 | 给我直接的处置建议 |
| 一线巡检员（资深） | 30% | 重复劳动多、写报告耗时 | 减少重复判定，自动出报告草稿 |
| 班组长 / 检修负责人 | 20% | 需要快速复盘 + 跨次对比 | 一键看资产历史与处置闭环 |
| 安全部 / 合规 | 10% | 需要可追溯、有依据 | 引用规程 + 引用案例必须可见 |

### 2.2 典型场景

1. 单图诊断：上传一张缺陷图 → 出诊断 + 处置建议
2. 报告生成：一次巡检 N 张图 → 出结构化报告草稿
3. 知识问答：用自然语言查规程、查历史
4. 跨次对比（v2 计划）：同一资产的时序变化

## 三、范围与不在范围

### 3.1 v1 范围

- VLM 视觉描述 + 多路文本 RAG
- 三类知识库：规程 / 历史案例 / 资产档案
- Streamlit Demo 前端 + 4 个核心 Tab
- RAGAS + 业务 KPI 评测

### 3.2 v1 不在范围（明确切割）

- 真实 YOLOv8 检测模型（v1.5）
- CLIP / BGE-VL 图像跨模态召回（v2.0）
- 知识图谱（v2.5）
- 多租户与权限隔离（v3.0）
- 边缘端部署（v3.0）

## 四、功能需求

### 4.1 缺陷诊断

| ID | 功能 | 优先级 | 验收标准 |
|---|---|---|---|
| F-DIA-01 | 支持 jpg / png / webp 图像上传 | P0 | 文件 ≤ 10MB 上传成功 |
| F-DIA-02 | VLM 视觉描述返回结构化 JSON | P0 | 包含 device_type / defects / keywords / image_quality 4 字段 |
| F-DIA-03 | 自动检索规程 Top-K 与案例 Top-K | P0 | 默认 K=8 / K=5 |

| F-DIA-05 | 当资产已选时附带历史巡检 | P1 | 倒序展示最近 5 条 |

| F-REP-03 | 支持下载 .md 文件 | P0 | 一键下载 |

### 4.3 知识问答

| ID | 功能 | 优先级 | 验收标准 |
|---|---|---|---|

### 4.4 评测体系

| ID | 功能 | 优先级 | 验收标准 |
|---|---|---|---|

## 五、非功能需求

| 维度 | 要求 |
|---|---|
| 性能 | 单图诊断端到端 < 30s（含 VLM）；问答 < 8s |
| 成本 | 单次诊断 ≤ 0.05 元（按 API 公开报价估算） |
| 可观测 | DEBUG 模式打印 query / chunks / 调用 token 数 |
| 数据合规 | 全部使用合成 / 公开数据，无 PII |
| 易用性 | 非工程师 5 分钟内可独立完成首次诊断 |

## 六、北极星指标与辅助指标

### 6.1 北极星指标

**工程师每份巡检报告的平均编辑次数下降率**
- 基线: 100%（人工从零撰写视为基准）
- v1 目标: 下降 ≥ 60%
- v2 目标: 下降 ≥ 80%

### 6.2 辅助指标

| 指标 | 基线 | v1 目标 |
|---|---|---|
| 单图诊断耗时 | 人工 4 分钟 | ≤ 30 秒 |
| 报告生成耗时 | 人工 90 分钟 | ≤ 5 分钟 |
| 诊断引用命中率（人工抽检） | — | ≥ 80% |
| 等级判定准确率 | — | ≥ 75% |

## 七、技术方案概要

详见 `architecture.md`。核心：

```
图像 → VLM 描述 → 多路召回（规程 + 案例 + 资产历史）→ RRF 融合 → LLM 诊断/报告
```

## 九、风险与对策

| 风险 | 概率 | 影响 | 对策 |
|---|---|---|---|
| VLM 误判工业缺陷细节 | 高 | 中 | 引入 YOLO 专项模型兜底（v1.5） |
| 大模型 API 不可用 / 限流 | 中 | 高 | 准备本地 7B 模型回退（v2） |
| 工程师不信任 AI 结论 | 中 | 高 | 强制引用溯源 + "答得不好"反馈按钮 |
| 数据泄露 | 低 | 极高 | 私有化部署 + 数据脱敏（v3） |
